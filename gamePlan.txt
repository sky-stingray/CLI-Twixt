🧠 High-Level Architecture

Your app will have 4 main modules, plus a CLI layer and persistent storage:

C-Unplugged
├── main.c                // CLI loop, command parser
├── song.c / song.h       // Song structure + file loading
├── album.c / album.h     // Album management (linked list)
├── playlist.c / playlist.h// Playlist logic (linked list + loop)
├── logger.c / logger.h   // Command logging
├── data/
│   ├── songs.txt         // Master song library
│   ├── albums/           // Folder for saved albums
│   └── log.txt           // Command log (persistent)

⚙️ Modules and Their Roles
1. Song Module (song.c/h)

Data Structure:

typedef struct Song {
    char title[100];
    char artist[100];
    char album[100];
    int duration; // in seconds
    struct Song *next;
} Song;

Functions:

    Song* loadSongsFromFile(const char *filename);

    void listSongs(Song *head);

    Song* findSong(Song *head, const char *title);

Persistence: Songs are read-only from a flat file (e.g., songs.txt).
2. Album Module (album.c/h)

Data Structure:

typedef struct Album {
    char name[100];
    Song *songs; // Linked list of songs
    struct Album *next;
} Album;

Functions:

    Album* loadAlbums(); → loads all saved albums from /data/albums/

    void saveAlbum(Album *album);

    void addSongToAlbum(Album *album, Song *song);

    void deleteSongFromAlbum(Album *album, const char *songTitle);

    void listAlbums(Album *head);

    void listSongsInAlbum(Album *album);

Persistence: Each album is saved in a separate file (e.g., data/albums/RockHits.txt).
3. Playlist Module (playlist.c/h)

Data Structure:

typedef struct PlaylistNode {
    Song *song;
    struct PlaylistNode *next;
    struct PlaylistNode *prev;
} PlaylistNode;

typedef struct Playlist {
    PlaylistNode *head;
    PlaylistNode *current;
} Playlist;

Functions:

    void addSongToPlaylist(Playlist *pl, Song *song);

    void addAlbumToPlaylist(Playlist *pl, Album *album);

    void playNext(Playlist *pl);

    void playPrev(Playlist *pl);

    void removeSongFromPlaylist(Playlist *pl, const char *title);

    void showCurrentSong(Playlist *pl);

Behavior:

    Playlist is circular: current->next wraps to head.

4. Logger Module (logger.c/h)

Functions:

    void logCommand(const char *command);

    void showLog();

Persistence: All logs appended to data/log.txt across runs.
5. CLI Module (main.c)

Handles:

    Parsing commands (using strcmp() or strncmp())

    Calling the right functions

    Displaying output in a clean, interactive way

Libraries to Use
Purpose	C Standard Library
File I/O	<stdio.h>, <stdlib.h>
Strings	<string.h>
Dynamic Memory	<stdlib.h>
Time (optional, for logs)	<time.h>
CLI Colors (optional)	ANSI escape codes
Directory management	<dirent.h> (for album folder)

No external libraries are strictly required — pure C standard is enough.
Data Persistence

    Songs: Loaded from a static file (e.g., songs.txt)

    Albums: Each album stored as text file, example:

    data/albums/MyRockAlbum.txt
    Hotel California|Eagles|Rock|390
    Stairway to Heaven|Led Zeppelin|Rock|480

    Logs: data/log.txt keeps every command entered by user.

🔁 Program Flow Summary

Start App
│
├─► Load Songs → Linked List
├─► Load Albums → Linked List
│
└─► CLI Loop:
     ├─ Parse user command
     ├─ Execute function (songs/albums/playlist)
     ├─ Log the command to file
     ├─ Display result
     ├─ Loop until "exit"
│
└─► Save modified albums on exit

🧩 Extra Tips

    Use linked lists for songs/albums/playlists for dynamic memory management.

    Handle file parsing with fgets() and strtok().

    Wrap I/O in utility functions for cleaner code.

    For a better UX, color CLI output using ANSI codes:

    printf("\033[1;32mSuccess!\033[0m\n");

    Use a Makefile to build the project modularly.
